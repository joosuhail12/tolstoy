# Quickstart Guide

This guide will help you create your first workflow in Tolstoy in under 5 minutes. We'll walk through creating a simple workflow that fetches data from an API and processes it.

<Info>
**Prerequisites:** You'll need a Tolstoy account and access to the API. If you don't have an account yet, [contact our team](mailto:support@tolstoy.dev) for access.
</Info>

## Step 1: Set Up Your Organization

Every workflow in Tolstoy belongs to an organization. Let's create your organization first:

<CodeGroup>

```bash cURL
curl -X POST 'https://api.tolstoy.dev/organizations' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "My Company",
    "settings": {
      "timezone": "America/New_York",
      "defaultRetryPolicy": {
        "maxRetries": 3,
        "backoffMultiplier": 2
      }
    }
  }'
```

```javascript TypeScript SDK
import { TolstoyClient } from '@tolstoy/sdk';

const client = new TolstoyClient({
  apiKey: 'YOUR_API_KEY',
  baseURL: 'https://api.tolstoy.dev'
});

const organization = await client.organizations.create({
  name: 'My Company',
  settings: {
    timezone: 'America/New_York',
    defaultRetryPolicy: {
      maxRetries: 3,
      backoffMultiplier: 2
    }
  }
});

console.log('Organization created:', organization.id);
```

```bash CLI
tolstoy orgs create \
  --name "My Company" \
  --timezone "America/New_York"
```

</CodeGroup>

## Step 2: Create Your First Tool

Tools in Tolstoy represent external services you want to integrate with. Let's create a tool for a REST API:

<CodeGroup>

```bash cURL
curl -X POST 'https://api.tolstoy.dev/tools' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "JSONPlaceholder API",
    "type": "api",
    "baseUrl": "https://jsonplaceholder.typicode.com",
    "authType": "none",
    "description": "Demo REST API for testing"
  }'
```

```javascript TypeScript SDK
const tool = await client.tools.create({
  name: 'JSONPlaceholder API',
  type: 'api',
  baseUrl: 'https://jsonplaceholder.typicode.com',
  authType: 'none',
  description: 'Demo REST API for testing'
});

console.log('Tool created:', tool.id);
```

</CodeGroup>

## Step 3: Create an Action

Actions are reusable workflow components. Let's create an action that fetches user data:

<CodeGroup>

```bash cURL
curl -X POST 'https://api.tolstoy.dev/actions' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "key": "fetch_user",
    "name": "Fetch User",
    "description": "Fetch user data from JSONPlaceholder",
    "toolId": "TOOL_ID_FROM_STEP_2",
    "method": "GET",
    "endpoint": "/users/{{userId}}",
    "inputSchema": {
      "type": "object",
      "properties": {
        "userId": {
          "type": "integer",
          "description": "User ID to fetch",
          "minimum": 1,
          "maximum": 10
        }
      },
      "required": ["userId"]
    }
  }'
```

```javascript TypeScript SDK
const action = await client.actions.create({
  key: 'fetch_user',
  name: 'Fetch User',
  description: 'Fetch user data from JSONPlaceholder',
  toolId: tool.id,
  method: 'GET',
  endpoint: '/users/{{userId}}',
  inputSchema: {
    type: 'object',
    properties: {
      userId: {
        type: 'integer',
        description: 'User ID to fetch',
        minimum: 1,
        maximum: 10
      }
    },
    required: ['userId']
  }
});

console.log('Action created:', action.key);
```

</CodeGroup>

## Step 4: Test Your Action

Before creating a workflow, let's test the action:

<CodeGroup>

```bash cURL
curl -X POST 'https://api.tolstoy.dev/actions/fetch_user/execute' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "inputs": {
      "userId": 1
    }
  }'
```

```javascript TypeScript SDK
const result = await client.actions.execute('fetch_user', {
  inputs: {
    userId: 1
  }
});

console.log('User data:', result.outputs);
```

</CodeGroup>

## Step 5: Create Your First Flow

Now let's create a flow (workflow) that uses our action:

<CodeGroup>

```bash cURL
curl -X POST 'https://api.tolstoy.dev/flows' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "User Info Workflow",
    "description": "Fetch and process user information",
    "steps": [
      {
        "id": "fetch_user",
        "name": "Fetch User Data", 
        "actionKey": "fetch_user",
        "inputs": {
          "userId": "{{ workflow.inputs.userId }}"
        }
      }
    ],
    "inputSchema": {
      "type": "object",
      "properties": {
        "userId": {
          "type": "integer",
          "description": "User ID to fetch",
          "default": 1
        }
      },
      "required": ["userId"]
    }
  }'
```

```javascript TypeScript SDK
const flow = await client.flows.create({
  name: 'User Info Workflow',
  description: 'Fetch and process user information',
  steps: [
    {
      id: 'fetch_user',
      name: 'Fetch User Data',
      actionKey: 'fetch_user',
      inputs: {
        userId: '{{ workflow.inputs.userId }}'
      }
    }
  ],
  inputSchema: {
    type: 'object',
    properties: {
      userId: {
        type: 'integer',
        description: 'User ID to fetch',
        default: 1
      }
    },
    required: ['userId']
  }
});

console.log('Flow created:', flow.id);
```

</CodeGroup>

## Step 6: Execute Your Flow

Now let's execute the workflow:

<CodeGroup>

```bash cURL
curl -X POST 'https://api.tolstoy.dev/flows/FLOW_ID/execute' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "inputs": {
      "userId": 2
    }
  }'
```

```javascript TypeScript SDK
const execution = await client.flows.execute(flow.id, {
  inputs: {
    userId: 2
  }
});

console.log('Execution started:', execution.id);

// Poll for completion
const result = await client.flows.waitForExecution(flow.id, execution.id);
console.log('Execution completed:', result.status);
```

</CodeGroup>
