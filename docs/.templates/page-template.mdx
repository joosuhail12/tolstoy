## Style Guide Notes

### Writing Style
- **Tone**: Professional but approachable
- **Voice**: Active voice preferred
- **Person**: Use "you" to address the reader directly
- **Tense**: Present tense for instructions, past tense for examples

### Formatting Guidelines
- **Headings**: Use sentence case (capitalize only first word and proper nouns)
- **Code**: Always use syntax highlighting with appropriate language tags
- **Links**: Use descriptive text, avoid "click here" or "read more"
- **Lists**: Use parallel structure and consistent formatting

### Content Requirements
- **Examples**: Every concept should have a practical example
- **Context**: Always explain why, not just how
- **Completeness**: Include error handling and edge cases
- **Testing**: Show how to verify implementations work

### SEO Considerations
- **Title**: Include primary keyword, keep under 60 characters
- **Description**: Meta description under 160 characters
- **Headings**: Use H2 and H3 tags with relevant keywords
- **Internal Links**: Link to related pages within the documentation

### Accessibility
- **Alt text**: Provide meaningful descriptions for images
- **Contrast**: Ensure text meets WCAG contrast requirements
- **Structure**: Use semantic HTML and proper heading hierarchy
- **Navigation**: Ensure all content is keyboard accessible
