### ‚ö†Ô∏è Breaking Changes

#### API Changes
- **Workflow Execution API**: Response format changed for parallel execution results
  ```json
  // Before
  {
    "result": "success",
    "output": "single_output"
  }
  
  // After
  {
    "result": "success",
    "outputs": {
      "step_1": "output_1",
      "step_2": "output_2"
    },
    "execution_summary": {
      "total_steps": 5,
      "successful_steps": 5,
      "parallel_groups": 2
    }
  }
  ```

- **Organization API**: Removed deprecated `legacy_settings` field
- **Integration API**: Changed authentication flow for OAuth integrations

#### Database Schema
- **Workflows Table**: Added `execution_mode` column (migration required)
- **Executions Table**: Split `output` into `outputs` JSONB column
- **Users Table**: Removed `beta_features` column (replaced with feature flags)

### üìä Migration Guide

#### Database Migrations
```bash
# Backup database
pg_dump tolstoy_production > backup_$(date +%Y%m%d).sql

# Run migration
npm run migrate:2.4.0

# Verify migration
npm run migrate:verify
```

#### API Client Updates
Update your API clients to handle the new execution response format:

```javascript
// Before
const result = await tolstoy.executeWorkflow(workflowId, input);
console.log(result.output);

// After  
const result = await tolstoy.executeWorkflow(workflowId, input);
console.log(result.outputs); // Now an object with step outputs
console.log(result.execution_summary); // New summary information
```

#### Configuration Changes
Update environment variables:
```bash
# New required variables
FEATURE_FLAGS_ENABLED=true
PARALLEL_EXECUTION_MAX_WORKERS=10

# Deprecated (will be removed in v3.0)
LEGACY_WORKFLOW_MODE=false
```
