## Quick Overview

```typescript
import { TolstoyClient } from '@joosuhail/tolstoy-sdk';

const client = new TolstoyClient(
  'https://tolstoy.getpullse.com',
  'your-org-id',
  'your-user-id'
);

// Execute workflows
const execution = await client.runFlow('flow_123', {
  userId: 'user_456',
  email: 'user@example.com'
});

// Manage integrations
const tool = await client.createTool({
  name: 'GitHub API',
  baseUrl: 'https://api.github.com',
  authType: 'bearer'
});

// Set up webhooks
const webhook = await client.createWebhook({
  name: 'Flow Monitor',
  url: 'https://your-app.com/webhooks',
  events: ['flow.completed', 'flow.failed']
});
```

## Architecture

The SDK provides two levels of access:

### 1. Helper Methods (Recommended)
Simplified methods for common operations with sensible defaults.

```typescript
// Simple workflow execution
const execution = await client.runFlow('flow_id', inputs);

// Easy tool creation
const tool = await client.createTool(toolConfig);
```

### 2. Raw API Access (Advanced)
Direct access to all generated endpoints for advanced use cases.

```typescript
// Advanced operations
const metrics = await client.raw.flows.flowsControllerGetMetrics('flow_id');
const health = await client.raw.health.healthControllerGetDetailedHealth();
```

## Multi-Tenant Architecture

Built for SaaS applications with organization-level isolation:

```typescript
// Different clients for different tenants
const clientA = new TolstoyClient(baseUrl, 'org_a', 'user_123');
const clientB = new TolstoyClient(baseUrl, 'org_b', 'user_456');

// Each client operates in its own tenant context
await clientA.runFlow('flow_id', inputs); // Isolated to org_a
await clientB.runFlow('flow_id', inputs); // Isolated to org_b
```
