# Tolstoy TypeScript SDK

The official TypeScript SDK for the Tolstoy workflow automation platform. Build powerful integrations with type safety and modern JavaScript/TypeScript patterns.

## Why Use the Tolstoy SDK?

- **🔒 Multi-tenant Ready**: Built-in support for organization and user context
- **📝 Full Type Safety**: Complete TypeScript support with IntelliSense
- **🎯 Simple & Powerful**: Helper methods for common tasks + raw API access
- **⚡ Production Ready**: Comprehensive error handling and testing
- **🔄 Always Current**: Auto-generated from OpenAPI specification

## Quick Overview

```typescript
import { TolstoyClient } from '@joosuhail/tolstoy-sdk';

const client = new TolstoyClient(
  'https://tolstoy.getpullse.com',
  'your-org-id',
  'your-user-id'
);

// Execute workflows
const execution = await client.runFlow('flow_123', {
  userId: 'user_456',
  email: 'user@example.com'
});

// Manage integrations
const tool = await client.createTool({
  name: 'GitHub API',
  baseUrl: 'https://api.github.com',
  authType: 'bearer'
});

// Set up webhooks
const webhook = await client.createWebhook({
  name: 'Flow Monitor',
  url: 'https://your-app.com/webhooks',
  events: ['flow.completed', 'flow.failed']
});
```

## Key Features

### ⚡ Workflow Automation
Execute and monitor workflows with full type safety. Support for both synchronous and asynchronous execution modes.

### 🔧 Tool Integrations
Create and manage connections to external APIs and services. Support for multiple authentication methods.

### 🔗 Event-Driven Architecture
Set up webhooks for real-time notifications about workflow events, user actions, and system changes.

### 📊 Complete API Coverage
Access to all **52 API endpoints** across health, organizations, users, tools, actions, flows, logs, and webhooks.

## Architecture

The SDK provides two levels of access:

### 1. Helper Methods (Recommended)
Simplified methods for common operations with sensible defaults.

```typescript
// Simple workflow execution
const execution = await client.runFlow('flow_id', inputs);

// Easy tool creation
const tool = await client.createTool(toolConfig);
```

### 2. Raw API Access (Advanced)
Direct access to all generated endpoints for advanced use cases.

```typescript
// Advanced operations
const metrics = await client.raw.flows.flowsControllerGetMetrics('flow_id');
const health = await client.raw.health.healthControllerGetDetailedHealth();
```

## Multi-Tenant Architecture

Built for SaaS applications with organization-level isolation:

```typescript
// Different clients for different tenants
const clientA = new TolstoyClient(baseUrl, 'org_a', 'user_123');
const clientB = new TolstoyClient(baseUrl, 'org_b', 'user_456');

// Each client operates in its own tenant context
await clientA.runFlow('flow_id', inputs); // Isolated to org_a
await clientB.runFlow('flow_id', inputs); // Isolated to org_b
```

## CLI Tool

In addition to the TypeScript SDK, Tolstoy provides a powerful command-line interface for managing authentication, executing actions, and automating workflows.

### 🖥️ Command-Line Interface
Perfect for scripts, CI/CD pipelines, and system administration:

```bash
# Configure tool authentication
tolstoy tool auth api-key --org my-org --tool github --key ghp_xxxxxxxxxxxx

# Execute actions directly
tolstoy actions:execute --org my-org --key send-email '{"to": "user@example.com", "subject": "Hello"}'

# Manage OAuth2 flows
tolstoy auth login --tool github --user user-123
```

**Key CLI Features:**
- 🔧 **Tool Configuration**: Set up API keys and OAuth2 credentials
- ⚡ **Action Execution**: Run individual actions with JSON inputs
- 🔐 **User Authentication**: Handle OAuth2 login flows
- 📝 **Template Management**: Import and manage workflow templates
- 🚀 **CI/CD Ready**: Perfect for automation and scripting
