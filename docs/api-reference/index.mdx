## Authentication

All API requests require these headers for multi-tenant access:

- **x-org-id**: Your organization ID
- **x-user-id**: Your user ID for request context

## Base URLs

- **Production**: `https://tolstoy.getpullse.com`
- **Development**: `http://localhost:3000`

## Rate Limits

- **General API**: 1000 requests per minute
- **Flow Execution**: 100 requests per minute  
- **Webhooks**: 500 requests per minute
- **Metrics**: 200 requests per minute

## Quick Start Examples

### 1. Create a Tool Integration
```bash
curl -X POST https://tolstoy.getpullse.com/tools \
  -H "x-org-id: your-org-id" \
  -H "x-user-id: your-user-id" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "GitHub API",
    "baseUrl": "https://api.github.com",
    "authType": "bearer"
  }'
```

### 2. Execute a Workflow
```bash
curl -X POST https://tolstoy.getpullse.com/flows/{flowId}/execute \
  -H "x-org-id: your-org-id" \
  -H "x-user-id: your-user-id" \
  -H "Content-Type: application/json" \
  -d '{
    "mode": "async",
    "inputs": {
      "email": "user@example.com"
    }
  }'
```

### 3. Monitor Execution
```bash
curl https://tolstoy.getpullse.com/flows/{flowId}/executions/{executionId} \
  -H "x-org-id: your-org-id" \
  -H "x-user-id: your-user-id"
```

## Error Handling

All API errors follow a consistent format:

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid input parameters",
    "details": {
      "field": "name", 
      "issue": "required"
    }
  },
  "meta": {
    "timestamp": "2024-01-15T10:30:00Z",
    "requestId": "req_abc123"
  }
}
```

### Common Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `VALIDATION_ERROR` | 400 | Invalid request parameters |
| `UNAUTHORIZED` | 401 | Missing or invalid authentication |
| `FORBIDDEN` | 403 | Insufficient permissions |
| `NOT_FOUND` | 404 | Resource not found |
| `CONFLICT` | 409 | Resource already exists |
| `RATE_LIMITED` | 429 | Too many requests |
| `INTERNAL_ERROR` | 500 | Server error |
