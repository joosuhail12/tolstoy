# Tools

Tools in Tolstoy represent external services and APIs that your workflows can integrate with. They provide standardized ways to authenticate, configure, and interact with third-party services, making it easy to build powerful automation workflows.

## What are Tools?

Tools are reusable integration components that abstract the complexity of working with external services. Each tool defines:

- **Authentication methods** - How to securely connect to the service
- **Configuration parameters** - Service-specific settings and options  
- **Available actions** - Operations that can be performed with the service
- **Data schemas** - Input/output formats for seamless integration

## Tool Configuration

### Setting Up a New Tool

<CodeGroup>
```bash CLI - Add Tool
# Add a new GitHub tool with OAuth
tolstoy tools add github \
  --name "Company GitHub" \
  --oauth-client-id "$GITHUB_CLIENT_ID" \
  --oauth-client-secret "$GITHUB_CLIENT_SECRET" \
  --config organization="your-org"

# List available tools
tolstoy tools list

# Test tool connectivity
tolstoy tools test github-company
```

```typescript TypeScript SDK
import { TolstoyClient } from '@tolstoy/sdk';

const client = new TolstoyClient({
  apiKey: 'your-api-key',
  organizationId: 'org_123'
});

// Add a new tool
const tool = await client.tools.create({
  type: 'github',
  name: 'Company GitHub',
  config: {
    baseUrl: 'https://api.github.com',
    organization: 'your-org',
    webhookSecret: 'webhook-secret'
  },
  oauth: {
    clientId: process.env.GITHUB_CLIENT_ID,
    clientSecret: process.env.GITHUB_CLIENT_SECRET,
    scopes: ['repo', 'admin:org']
  }
});

// Test tool connection
const testResult = await client.tools.test(tool.id);
```


## Custom Tools

### Building Custom Tool Integrations

For services not covered by built-in tools, you can create custom integrations:

<CodeGroup>
```typescript Custom Tool Definition
interface CustomToolConfig {
  baseUrl: string;
  apiVersion: string;
  timeout: number;
  retries: number;
}

interface CustomToolCredentials {
  apiKey: string;
  secretKey?: string;
}

const customTool = await client.tools.createCustom({
  type: 'custom-api',
  name: 'Internal CRM API',
  description: 'Integration with our internal CRM system',
  config: {
    baseUrl: 'https://crm.company.com/api',
    apiVersion: 'v2',
    timeout: 30000,
    retries: 3
  } as CustomToolConfig,
  credentials: {
    apiKey: process.env.CRM_API_KEY,
    secretKey: process.env.CRM_SECRET_KEY
  } as CustomToolCredentials,
  actions: [
    {
      name: 'create-contact',
      description: 'Create a new contact in CRM',
      inputSchema: {
        type: 'object',
        properties: {
          name: { type: 'string', required: true },
          email: { type: 'string', format: 'email', required: true },
          phone: { type: 'string' },
          company: { type: 'string' }
        }
      },
      outputSchema: {
        type: 'object',
        properties: {
          id: { type: 'string' },
          status: { type: 'string' }
        }
      }
    }
  ]
});
```

```typescript Custom Action Implementation
// Custom action execution logic
const customActionHandler = async (config: any, context: any) => {
  const { baseUrl, apiVersion } = config.tool.config;
  const { apiKey } = config.tool.credentials;
  const { name, email, phone, company } = config.input;

  const response = await fetch(`${baseUrl}/${apiVersion}/contacts`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name,
      email,
      phone,
      company,
      source: 'tolstoy-automation'
    })
  });

  if (!response.ok) {
    throw new Error(`CRM API error: ${response.statusText}`);
  }

  return await response.json();
};
```
</CodeGroup>

## Security and Best Practices

### Credential Management

<CardGroup cols={2}>
  <Card title="Secure Storage" icon="vault">
    All credentials are encrypted at rest using AWS KMS and stored securely in AWS Secrets Manager
  </Card>
  
  <Card title="Rotation" icon="rotate">
    Implement regular credential rotation for API keys and OAuth token refresh
  </Card>
  
  <Card title="Least Privilege" icon="lock">
    Configure tools with minimal required permissions and scopes
  </Card>
  
  <Card title="Audit Trail" icon="list-check">
    All tool usage is logged for security and compliance purposes
  </Card>
</CardGroup>

### OAuth Security

```typescript
// OAuth best practices
const oauthConfig = {
  // Use PKCE for public clients
  usePkce: true,
  
  // Validate state parameter
  validateState: true,
  
  // Minimal required scopes
  scopes: ['read:user', 'user:email'], // Not 'admin:org' unless needed
  
  // Secure redirect URI
  redirectUri: 'https://app.tolstoy.dev/tools/oauth/callback',
  
  // Token refresh settings
  refreshThreshold: 300, // Refresh 5 minutes before expiry
  maxRefreshAttempts: 3
};
```

```json Health Check Response
  "lastCheck": "2024-01-20T14:30:00Z"
```

### Common Issues and Solutions

<Accordion>
  <AccordionItem title="OAuth Token Expired">
    **Symptoms:** Authentication errors, 401 responses
    
    **Solutions:**
    - Enable automatic token refresh
    - Check token expiry and refresh manually
    - Verify OAuth configuration and scopes
    
    ```bash
    # Refresh OAuth token
    tolstoy tools refresh-oauth github-company
    ```
  </AccordionItem>
  
  <AccordionItem title="Rate Limit Exceeded">
    **Symptoms:** 429 responses, throttling errors
    
    **Solutions:**
    - Implement exponential backoff
    - Reduce request frequency
    - Use request queuing
    - Consider upgrading API plan
    
    ```typescript
    // Configure rate limiting
    await client.tools.update('github-company', {
      rateLimiting: {
        enabled: true,
        requestsPerMinute: 60,
        burstLimit: 10
      }
    });
    ```
  </AccordionItem>
  
  <AccordionItem title="Connection Timeouts">
    **Symptoms:** Timeout errors, slow responses
    
    **Solutions:**
    - Increase timeout settings
    - Check network connectivity
    - Use connection pooling
    - Implement circuit breakers
    
    ```typescript
    // Update timeout configuration
    await client.tools.update('api-service', {
      config: {
        timeout: 30000,
        retries: 3,
        retryDelay: 1000
      }
    });
    ```
  </AccordionItem>
</Accordion>
