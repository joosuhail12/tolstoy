## Overview

Lists all actions in your organization. Actions are displayed in reverse chronological order (newest first) by default. Use query parameters to filter, sort, and paginate results.

This endpoint is essential for:
- Browsing available actions for workflow building
- Monitoring action inventory and usage
- Searching for specific actions by name or functionality
- Auditing action configurations and versions

## Authentication

<ParamField header="x-org-id" type="string" required>
  Your organization ID - determines which organization's actions to retrieve
</ParamField>

<ParamField header="x-user-id" type="string" required>
  Your user ID - used for permission checks and audit logging
</ParamField>

## Query Parameters

### Pagination

<ParamField query="page" type="integer" default="1">
  Page number for pagination (1-based)
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of actions per page (maximum 100)
</ParamField>

### Filtering

<ParamField query="search" type="string">
  Search term to filter actions by name or key
</ParamField>

<ParamField query="toolId" type="string">
  Filter actions by specific tool ID
</ParamField>

<ParamField query="method" type="string">
  Filter by HTTP method
  
  **Options:** `GET`, `POST`, `PUT`, `PATCH`, `DELETE`
</ParamField>

<ParamField query="status" type="string">
  Filter by action status
  
  **Options:** `active`, `inactive`, `draft`
</ParamField>

<ParamField query="createdAfter" type="string">
  ISO 8601 timestamp - only return actions created after this date
</ParamField>

<ParamField query="createdBefore" type="string">
  ISO 8601 timestamp - only return actions created before this date
</ParamField>

### Sorting

<ParamField query="sort" type="string" default="createdAt">
  Field to sort by
  
  **Options:** `name`, `key`, `createdAt`, `updatedAt`, `version`
</ParamField>

<ParamField query="order" type="string" default="desc">
  Sort direction
  
  **Options:** `asc`, `desc`
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET "https://tolstoy.getpullse.com/actions?search=stripe&limit=10" \
  -H "x-org-id: org_abc123def456" \
  -H "x-user-id: user_xyz789abc"
```

```javascript JavaScript SDK
import { TolstoyClient } from '@tolstoy/sdk';

const tolstoy = new TolstoyClient({
  orgId: 'org_abc123def456',
  userId: 'user_xyz789abc'
});

const response = await tolstoy.actions.list({
  search: 'stripe',
  limit: 10,
  sort: 'name',
  order: 'asc'
});

console.log(`Found ${response.data.length} actions`);
response.data.forEach(action => {

});
```

```python Python SDK
from tolstoy import TolstoyClient

client = TolstoyClient(
    org_id='org_abc123def456',
    user_id='user_xyz789abc'
)

response = client.actions.list(
    search='stripe',
    limit=10,
    sort='name',
    order='asc'
)

print(f"Found {len(response.data)} actions")
for action in response.data:
    print(f"- {action.name} ({action.method} {action.endpoint})")
```

```go Go SDK
package main

import (
    "context"
    "fmt"
    "log"

    "github.com/tolstoy/go-sdk/tolstoy"
)

func main() {
    client, err := tolstoy.NewClient(&tolstoy.Config{
        APIKey:         "your-api-key",
        OrganizationID: "org_abc123def456",
        UserID:         "user_xyz789abc",
    })
    if err != nil {
        log.Fatalf("Failed to create client: %v", err)
    }
    defer client.Close()

    ctx := context.Background()
    
    response, err := client.Actions.List(ctx, &tolstoy.ListActionsRequest{
        Search: "stripe",
        Limit:  10,
        Sort:   "name",
        Order:  "asc",
    })
    if err != nil {
        log.Fatalf("Failed to list actions: %v", err)
    }

    fmt.Printf("Found %d actions\n", len(response.Data))
    for _, action := range response.Data {
        fmt.Printf("- %s (%s %s)\n", action.Name, action.Method, action.Endpoint)
    }
}
```
</CodeGroup>

## Filtering Examples

### Search by Name or Key

```bash
GET /actions?search=customer
```

Returns all actions with "customer" in their name or key.

### Filter by Tool

```bash
GET /actions?toolId=tool_stripe_api&sort=executionCount&order=desc
```

Returns all Stripe actions sorted by execution count (most used first).

### Filter by HTTP Method

```bash
GET /actions?method=POST&status=active
```

Returns all active POST actions.

### Date Range Filtering

```bash
GET /actions?createdAfter=2024-01-01T00:00:00Z&createdBefore=2024-01-31T23:59:59Z
```

Returns actions created in January 2024.

## Performance Tips

<AccordionGroup>
  <Accordion title="Pagination Best Practices">
    - Use reasonable page sizes (10-50 items)
    - Cache results when possible to avoid repeated requests
    - Consider using `hasNext` and `hasPrev` for navigation UI
    - For large datasets, use filtering to narrow results
  </Accordion>
  
  <Accordion title="Efficient Filtering">
    - Use specific filters (toolId, method) instead of broad searches
    - Combine filters to reduce result sets (e.g., `toolId` + `status`)
    - Use date range filtering for time-based queries
    - Consider caching frequently accessed filter combinations
  </Accordion>
  
  <Accordion title="Sorting Strategy">
    - Default sort (createdAt desc) is most efficient
    - Sorting by `executionCount` helps identify popular actions
    - Use `name` sort for user-facing displays
    - Avoid frequent sort changes to leverage caching
  </Accordion>
</AccordionGroup>
