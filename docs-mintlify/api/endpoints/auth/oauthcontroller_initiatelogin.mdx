# Initiate OAuth2 login for a user

Redirects the user to the OAuth provider authorization page. The user must grant permission, after which they will be redirected back to the callback endpoint.

## Authentication

<ParamField header="x-org-id" type="string" required>
  Your organization ID
</ParamField>

<ParamField header="x-user-id" type="string" required>
  Your user ID
</ParamField>

<ParamField path="toolId" type="string" required>
  Tool unique identifier (database ID)
</ParamField>

<ParamField query="userId" type="string" required>
  User ID for whom to initiate OAuth
</ParamField>

<ParamField header="X-Org-ID" type="string" required>
  Organization identifier
</ParamField>



## Examples

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3000/auth/{toolId}/login" \
  -H "Content-Type: application/json" \
  -H "x-org-id: org_abc123def456" \
  -H "x-user-id: user_xyz789abc"
```

```javascript JavaScript SDK
import { TolstoyClient } from '@tolstoy/sdk';

const tolstoy = new TolstoyClient({
  orgId: 'org_abc123def456',
  userId: 'user_xyz789abc'
});

const result = await tolstoy.auth.oauthcontroller_initiatelogin();
console.log(result);
```
</CodeGroup>

## Responses

Redirects to OAuth provider authorization URL
