## Overview

The Create Organization endpoint allows you to create a new organization with specified settings, billing configuration, and initial user assignments. This is typically used during onboarding or when expanding to multi-organization setups.

## Authentication

<ParamField header="authorization" type="string" required>
  Bearer token for API authentication
</ParamField>

## Post-Creation Setup

After creating an organization, you typically want to:

1. **Configure Tool Integrations**: Set up authentication for external tools
2. **Create Workflows**: Build your first automation workflows
3. **Set Up Webhooks**: Configure event notifications
4. **Invite Additional Users**: Add team members with appropriate roles
5. **Configure API Keys**: Create additional API keys with specific permissions

## Organization Slug Rules

Organization slugs must:
- Be 3-50 characters long
- Contain only lowercase letters, numbers, and hyphens
- Start and end with a letter or number
- Not contain consecutive hyphens
- Be globally unique across all Tolstoy organizations
