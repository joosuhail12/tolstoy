## Overview

The Update Organization endpoint allows you to modify organization settings, billing information, user permissions, and other configuration options. This is typically used for ongoing organization management and settings updates.

## Authentication

<ParamField header="x-org-id" type="string" required>
  Organization ID for multi-tenant access control
</ParamField>

<ParamField header="x-user-id" type="string" required>
  User ID of the requesting user for authorization
</ParamField>

<ParamField header="authorization" type="string" required>
  Bearer token for API authentication
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the organization to update
</ParamField>

## Error Responses

<ResponseField name="400" type="object">
  Bad Request - Invalid data or validation errors
</ResponseField>

<ResponseField name="403" type="object">
  Forbidden - Insufficient permissions to update this organization
</ResponseField>

<ResponseField name="404" type="object">
  Not Found - Organization does not exist
</ResponseField>

<ResponseField name="422" type="object">
  Unprocessable Entity - Plan downgrade would exceed current usage
  
  <Expandable title="Plan Downgrade Error">
    <ResponseField name="error" type="string">
      "PLAN_DOWNGRADE_BLOCKED"
    </ResponseField>
    
    <ResponseField name="message" type="string">
      "Cannot downgrade plan due to current usage"
    </ResponseField>
    
    <ResponseField name="currentUsage" type="object">
      Current usage that prevents downgrade
    </ResponseField>
    
    <ResponseField name="targetLimits" type="object">
      Limits for the requested plan
    </ResponseField>
  </Expandable>
</ResponseField>

    headers={
        "Authorization": f"Bearer {api_token}",

if response.status_code == 200:

## Plan Changes

When updating the billing plan:

- **Upgrades**: Take effect immediately with prorated billing
- **Downgrades**: Require validation that current usage fits within new limits
- **Enterprise**: Contact support for custom configurations
