# Tools API

Tools in Tolstoy represent external API services that you want to integrate with. Each tool defines connection details, authentication methods, and serves as the foundation for creating actions.

## What are Tools?

Tools provide:
- **Base URL Configuration**: The root URL for all API calls to the external service
- **Authentication Setup**: How to authenticate with the external API (API keys, OAuth, etc.)
- **Connection Management**: Secure storage of credentials and connection details
- **Action Foundation**: The basis for creating reusable actions that call the external API

## Authentication Methods

Tools support various authentication methods:

<Tabs>
  <Tab title="API Key">
    Simple API key authentication in headers or query parameters
    ```json
    {
      "authType": "api_key",
      "authConfig": {
        "keyLocation": "header",
        "keyName": "X-API-Key"
      }
    }
    ```
  </Tab>
  
  <Tab title="Bearer Token">
    Bearer token authentication in Authorization header
    ```json
    {
      "authType": "bearer_token",
      "authConfig": {
        "tokenPrefix": "Bearer"
      }
    }
    ```
  </Tab>
  
  <Tab title="OAuth 2.0">
    OAuth 2.0 flow for user authorization
    ```json
    {
      "authType": "oauth2",
      "authConfig": {
        "grantType": "authorization_code",
        "authUrl": "https://api.service.com/oauth/authorize",
        "tokenUrl": "https://api.service.com/oauth/token"
      }
    }
    ```
  </Tab>
  
  <Tab title="Basic Auth">
    Username and password authentication
    ```json
    {
      "authType": "basic_auth",
      "authConfig": {
        "usernameField": "username",
        "passwordField": "password"
      }
    }
    ```
  </Tab>
</Tabs>

## Tool States

Tools can be in different states:

- **Active**: Ready for use in actions and workflows
- **Inactive**: Temporarily disabled, actions won't execute
- **Testing**: Under development, only accessible to creators
- **Error**: Connection issues or authentication failures

## Best Practices

<AccordionGroup>
  <Accordion title="Naming Conventions">
    - Use clear, descriptive names (e.g., "Stripe Production API")
    - Include environment when applicable (Production, Staging, Development)
    - Avoid generic names like "API" or "Service"
    - Be consistent across similar tools
  </Accordion>
  
  <Accordion title="Security">
    - Use separate tools for different environments
    - Regularly rotate API keys and credentials  
    - Implement least-privilege access for API keys
    - Monitor tool usage and authentication failures
  </Accordion>
  
  <Accordion title="Organization">
    - Group related tools by service or function
    - Document tool purposes and use cases
    - Test tool connections after setup
    - Keep tool configurations up to date
  </Accordion>
</AccordionGroup>
