# Tools API

Configure external API integrations and authentication

## Endpoints

<CardGroup cols={2}>
  <Card title="Create Tool" icon="api" href="/api/endpoints/tools/toolscontroller_create">
    Register a new external tool for use in workflows
  </Card>
  <Card title="List Tools" icon="api" href="/api/endpoints/tools/toolscontroller_findall">
    Get all registered tools for the organization
  </Card>
  <Card title="Get Tool" icon="api" href="/api/endpoints/tools/toolscontroller_findone">
    Get a specific tool by ID
  </Card>
  <Card title="Update Tool" icon="api" href="/api/endpoints/tools/toolscontroller_update">
    Update tool configuration
  </Card>
  <Card title="Delete Tool" icon="api" href="/api/endpoints/tools/toolscontroller_remove">
    Delete a tool permanently (will affect workflows using this tool)
  </Card>
  <Card title="Store Tool Credentials" icon="api" href="/api/endpoints/tools/toolsecretscontroller_storecredentials">
    Securely store encrypted credentials for an external tool. These credentials will be used when the tool is executed in workflows.
  </Card>
  <Card title="Get Tool Credentials" icon="api" href="/api/endpoints/tools/toolsecretscontroller_getcredentials">
    Retrieve stored credentials for a tool. By default returns masked values for security. Use unmask=true to get actual values (use carefully).
  </Card>
  <Card title="Delete Tool Credentials" icon="api" href="/api/endpoints/tools/toolsecretscontroller_deletecredentials">
    Permanently delete stored credentials for a tool. This will affect any workflows that use this tool.
  </Card>
  <Card title="List Tools with Credentials" icon="api" href="/api/endpoints/tools/toolsecretslistcontroller_listtoolswithcredentials">
    Get all tools in the organization with their credential status. Shows which tools have credentials stored and which ones need setup.
  </Card>
  <Card title="Create or update organization-level auth configuration for a tool" icon="api" href="/api/endpoints/tools/toolauthcontroller_upsert">
    Configures authentication settings (API Key or OAuth2) for a specific tool within an organization. API Key auth requires headerName and headerValue. OAuth2 auth requires clientId, clientSecret, and accessToken. A default callback URL is automatically added for OAuth2 configurations.
  </Card>
  <Card title="Fetch auth configuration(s) for a tool" icon="api" href="/api/endpoints/tools/toolauthcontroller_get">
    Retrieves authentication configuration(s) for a specific tool within an organization. Use configName query parameter to get a specific configuration, or omit to list all configurations.
  </Card>
  <Card title="Delete auth configuration for a tool" icon="api" href="/api/endpoints/tools/toolauthcontroller_remove">
    Removes authentication configuration(s) for a specific tool within an organization. Use configName query parameter to delete a specific configuration, or use deleteAll=true to delete all configurations.
  </Card>
</CardGroup>
