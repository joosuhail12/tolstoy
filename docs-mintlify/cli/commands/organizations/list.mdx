# tolstoy organizations list

Display all Tolstoy organizations you have access to, along with their details, membership status, and permissions.

## Usage

```bash
tolstoy organizations list [OPTIONS]
```

## Description

The `tolstoy organizations list` command shows all organizations associated with your account. This includes organizations where you're a member, admin, or owner, along with relevant metadata and access levels.

## Basic Usage

```bash
# List all organizations
tolstoy organizations list

# List with detailed information
tolstoy organizations list --detailed

# Filter by role
tolstoy organizations list --role admin
```

## Options

<ParamField path="--format, -f" type="string">
  Output format for the organization list
  
  **Options**: `table`, `json`, `yaml`, `csv`  
  **Default**: `table`
  
  ```bash
  tolstoy organizations list --format json
  ```
</ParamField>

<ParamField path="--detailed, -d" type="boolean">
  Show detailed information including member counts, created dates, and settings
  
  ```bash
  tolstoy organizations list --detailed
  ```
</ParamField>

<ParamField path="--role, -r" type="string">
  Filter organizations by your role
  
  **Options**: `owner`, `admin`, `member`, `viewer`
  
  ```bash
  tolstoy organizations list --role admin
  ```
</ParamField>

<ParamField path="--status" type="string">
  Filter organizations by status
  
  **Options**: `active`, `suspended`, `archived`
  
  ```bash
  tolstoy organizations list --status active
  ```
</ParamField>

<ParamField path="--search, -s" type="string">
  Search organizations by name or description
  
  ```bash
  tolstoy organizations list --search "acme"
  ```
</ParamField>

<ParamField path="--limit" type="number">
  Maximum number of organizations to display
  
  **Default**: `50`
  
  ```bash
  tolstoy organizations list --limit 10
  ```
</ParamField>

<ParamField path="--sort" type="string">
  Sort organizations by field
  
  **Options**: `name`, `created`, `updated`, `member_count`  
  **Default**: `name`
  
  ```bash
  tolstoy organizations list --sort created --reverse
  ```
</ParamField>

<ParamField path="--reverse" type="boolean">
  Reverse the sort order
  
  ```bash
  tolstoy organizations list --sort name --reverse
  ```
</ParamField>

<ParamField path="--fields" type="string">
  Specify which fields to display (comma-separated)
  
  ```bash
  tolstoy organizations list --fields "id,name,role,member_count"
  ```
</ParamField>

## Filtering and Searching

### Filter by Role

```bash
# Show only organizations where you're an owner
tolstoy organizations list --role owner

# Show admin and owner organizations
tolstoy organizations list --role admin,owner
```

### Search Organizations

```bash
# Search by name
tolstoy organizations list --search "acme"

# Search in description
tolstoy organizations list --search "development"

# Case-insensitive search
tolstoy organizations list --search "MARKETING"
```

### Status Filtering

```bash
# Active organizations only
tolstoy organizations list --status active

# Include suspended organizations
tolstoy organizations list --status active,suspended
```


## Role-Based Access

### Understanding Roles

<AccordionGroup>
  <Accordion title="Owner">
    **Permissions**:
    - Full access to organization settings
    - Billing and subscription management
    - Member management and role assignment
    - Delete organization
    - All admin and member permissions
    
    **Indicators**: 
    - Crown icon in CLI output
    - "owner" role badge
  </Accordion>
  
  <Accordion title="Admin">
    **Permissions**:
    - Manage workflows, tools, and actions
    - Invite and remove members
    - View organization analytics
    - Configure integrations
    - All member permissions
    
    **Indicators**: 
    - Shield icon in CLI output
    - "admin" role badge
  </Accordion>
  
  <Accordion title="Member">
    **Permissions**:
    - Create and modify own workflows
    - View organization workflows
    - Execute workflows
    - Access organization tools
    
    **Indicators**: 
    - User icon in CLI output
    - "member" role badge
  </Accordion>
  
  <Accordion title="Viewer">
    **Permissions**:
    - Read-only access to workflows
    - View execution history
    - Access organization directory
    
    **Indicators**: 
    - Eye icon in CLI output  
    - "viewer" role badge
  </Accordion>
</AccordionGroup>

## Scripting and Automation

### Bash Scripts

```bash
#!/bin/bash

# Get all organization IDs
ORG_IDS=$(tolstoy organizations list --format json | jq -r '.[].id')

# Process each organization
for org_id in $ORG_IDS; do
  echo "Processing organization: $org_id"
  tolstoy flows list --org "$org_id" --format json > "flows-${org_id}.json"
done
```

### PowerShell Scripts

```powershell
# Get organizations as PowerShell objects
$orgs = tolstoy organizations list --format json | ConvertFrom-Json

# Filter and process
$orgs | Where-Object { $_.role -eq "admin" } | ForEach-Object {
    Write-Host "Admin access to: $($_.name)"
    & tolstoy workflows list --org $_.id
}
```

### Python Integration

```python
import subprocess
import json

# Get organizations
result = subprocess.run(
    ['tolstoy', 'organizations', 'list', '--format', 'json'],
    capture_output=True, text=True
)

organizations = json.loads(result.stdout)

# Find organizations by criteria
admin_orgs = [org for org in organizations if org['role'] == 'admin']
large_orgs = [org for org in organizations if org['member_count'] > 10]

print(f"You are admin in {len(admin_orgs)} organizations")
print(f"Found {len(large_orgs)} organizations with >10 members")
```

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="No Organizations Listed">
    **Cause**: Not authenticated or no organization access
    
    **Solutions**:
    ```bash
    # Check authentication status
    tolstoy auth status
    
    # Re-authenticate
    tolstoy auth login
    
    # Verify organization access
    tolstoy auth whoami
    ```
  </Accordion>
  
  <Accordion title="Missing Organizations">
    **Cause**: Organizations may be suspended or you've lost access
    
    **Solutions**:
    ```bash
    # Include all statuses
    tolstoy organizations list --status active,suspended,archived
    
    # Check with organization admin
    # Contact support if access was revoked unexpectedly
    ```
  </Accordion>
  
  <Accordion title="Permission Denied">
    **Error**: `insufficient permissions to list organizations`
    
    **Solutions**:
    ```bash
    # Check current user permissions
    tolstoy auth whoami
    
    # Ensure you're using correct account
    tolstoy auth login --force
    ```
  </Accordion>
</AccordionGroup>

### Audit and Reporting

```bash

tolstoy organizations list --detailed --format json > org_report.json

# Count organizations by role
tolstoy organizations list --format json | \
  jq 'group_by(.role) | map({role: .[0].role, count: length})'

# Find high-usage organizations
tolstoy organizations list --format json | \
  jq '.[] | select(.usage.api_calls_month > 1000000) | .name'
```

### Bulk Operations

```bash
# Export all organization data
mkdir -p org_exports
tolstoy organizations list --format json | \
  jq -r '.[].id' | \
  while read org_id; do
    tolstoy flows list --org "$org_id" --format json > "org_exports/flows_${org_id}.json"
    tolstoy tools list --org "$org_id" --format json > "org_exports/tools_${org_id}.json"
  done
```
